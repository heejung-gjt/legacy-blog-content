예외처리

사용자가 없는 id pwd 를 입력할때처럼 예외적인 상황으로 발생하는 error 를 예외처리 해줌으로써 정상적인 종료가 가능하다. try ~ except ~ finally 구문을 사용한다

  • 📌 login 예외처리 해주기

  • 📌 board 예외처리 해주기

게시글이 없는 url을 입력했을 경우를 예외처리 해준다

  1. from django.http import Http404 : 내용을 찾을수 없을때 내는 페이지 404에러 페이지
  2. except Board.DoesNotExist: raise Http404(‘게시글을 찾을 수 없습니다’) : 없는 pk 이면, 즉 게시글이면 http404 에러페이지가 출력된다
board/views.py
def board_detail(request, pk):
    try:
        board = Board.objects.get(pk=pk)
        
    except Board.DoesNotExist:
        raise Http404('게시글을 찾을 수 없습니다')
    
    return render(request, 'board_detail.html',{'board':board})
