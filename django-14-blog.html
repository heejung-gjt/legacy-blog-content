django의 User 모델 확장 AbstractUser

기존 장고의 user필드에 있는 기능외에 필드를 추가하여 커스터마이징 하는것이다
총 4가지의 방법이 있다. 그중 AbstractUser는 user model을 그대로 가져다 쓰고 필드만 재정의 할때 사용한다

User model

유저 모델에 있는 디폴트 필드

  • username
  • password
  • first_name
  • last_name
  • email
  • last_login : 로그인한 마지막 시간
  • date_joined : 계정이 생성된 날짜


  • AbstractUser를 사용하여 model을 만들어보자

    가장 먼저 AbstractUser를 import해주어야 한다.

from django.db import models
from django.contrib.auth.models import AbstractUser,BaseUserManager
# Create your models here.

GENDER_CHOICES = (
    (0, 'Male'),
    (1, 'Female'),
    (2, 'Not to disclose')
)

class UserManager(BaseUserManager): #user에서 사용하기 위한 UserManager생성
    
    def _create_user(self, email, username, password, gender=2, **extra_fields):
        if not email:
            raise ValueError('The given email must be set')
        email = self.normalize_email(email) #이메일 주소를 소문자로 변환하는 과정을 거친 뒤 정장
        username = self.model.normalize_username(username)
        user = self.model(email=email, username=username, gender=gender, **extra_fields) #사용자 모델 객체 생성 
        user.set_password(password) # user pwd는 django에서 제공해주는 해시화 과정을 거쳐 저장됨
        user.save(using=self._db) # 실제로 db에 사용자 정보를 저장한다 
        return user

    def create_user(self, email, username='', password=None, **extra_fields): #일반 유저일경우
        extra_fields.setdefault('is_staff', False)
        extra_fields.setdefault('is_superuser', False)
        return self._create_user(email, username, password, **extra_fields)

    def create_superuser(self, email, password, **extra_fields):
        extra_fields.setdefault('is_staff', True)
        extra_fields.setdefault('is_superuser', True)

        if extra_fields.get('is_staff') is not True:
            raise ValueError('Superuser must have is_staff=True.')
        if extra_fields.get('is_superuser') is not True:
            raise ValueError('Superuser must have is_superuser=True.')
        return self._create_user(email, 'blogs/like_section.html',  password, **extra_fields)


class User(AbstractUser):
    email = models.EmailField(verbose_name='email', max_length=255, unique=True)
    username = models.CharField(max_length=30)
    gender = models.SmallIntegerField(choices=GENDER_CHOICES)
    objects = UserManager()
    USERNAME_FIELD = 'email'
    REQUIRED_FIELDS = [] # 필수로 받고 싶은 필드들 넣기 원래 소스 코드엔 email필드가 들어가지만 우리는 로그인을 이메일로 하니깐..

    def __str__(self):
        return "<%d %s>" % (self.pk, self.email)


  • 변경한 model로 DB를 변경한다

    mirgrate하기 전에 settings.py에서 AUTH_USER_MODEL = 'users.User'을 넣어주고 users의 admin페이지에 User 모델을 추가하기 위해서 다음 코드를 작성한다

users/admin.py
from django.contrib import admin
from .models import User
# Register your models here.
admin.site.register(User) #admin 사이트에 User모델 추가한다
