Django - model ForeignKeyField

model을 설계할때 생기는 모델 간의 관계를 1:1, 1:N, N:N으로 정의 할 수 있다

ForeignKey

ForeignKey는 1:n 관계를 말한다. 예를 들어보자면 주문 - 사용자를 생각해 볼 수 있다. 한명의 사용자가 여러개의 주문을 할 수 있게 되므로 foreignkey를 만족한다. foreignkey를 사용할때는 on_delete값을 꼭 설정해주어야 한다

on_delete 설정

1:N에서 1인쪽의 데이터가 삭제되었을 경우 N의 데이터를 어떻게 처리할 것인지에 대한 옵션이다

  • CASCADE
    1:N에서 1과 연결되는 모든 N들의 데이터를 삭제한다

  • PROTECT
    1:N에서 1이 삭제되지 않도록 막아준다

  • SET_NULL
    1:N에서 1이 삭제될때 연결되는 N들의 데이터값을 NULL로 바꾼다 단, null=True일때만 가능하다

  • SET_DEFAULT
    1:N에서 1이 삭제될때 N들의 데이터값을 default값으로 바꾼다 단, default값이 존재할때만 가능하다

  • SET
    1:N에서 1이 삭제될때 N의 데이터값을 대체할 값이나 함수로 지정해준다

  • DO_NOTHING
    1:N에서 1이 삭제되도 아무것도 하지 않는다. 오류가 발생할 수 있다


(EX)

① user앱 안에 있는 User model을 사용하여 user가 삭제되면 user와 연결된 모든 model이 연쇄적으로 삭제되게 설정하였다
② product앱 안에 있는 Product model을 사용하여 product가 삭제되면 연결된 모든 model이 연쇄적으로 삭제되게 설정하였다

user = models.ForeignKey('user.User', on_delete=models.CASCADE, verbose_name='사용자')  - ①
