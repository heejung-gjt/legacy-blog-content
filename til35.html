210413-TIL (웹동작 방식, CLI, 스크래핑 실습)

웹이 동작하는 방식

클라이언트가 요청을 보내면 서버가 해당 요청을 받아 응답해준다 client는 서비스를 요청하는 프로그램이고 server는 클라이언트의 요청에 대한 응답을 주는 프로그램이다


프로토콜

컴퓨터나 네트워크 장비가 서로 통신하기 위해 미리 정해놓은 약속,규약이다 사람과 사람이 서로 통신할때 이해할수 있는 공통된 언어를 사용하는 것처럼 컴퓨터와 컴퓨터 사이에도 서로 이해할 수 있는 공통된 언어를 사용하는데 이것을 프로토콜이라고 한다.


HTTP 프로토콜

  • 인터넷 통신을 위해 사용되는 프로토콜이다
  • HTTP프로토콜은 인터넷상에서 데이터를 주고 받기 위한 서버/클라이언트 모델을 따르는 프로토콜이다
  • HTTP프로토콜은 HTML문서와 같은 리소스들을 가져올 수 있도록 해준다
  • HTTP는 서버/클라이언트 모델이다
  • HTTP는 기본적으로 평문 데이터 전송을 원칙으로 하기 때문에 프라이버시가 오가는 서비스에서 사용하기는 어려움이 있다(HTTPS사용)


HTTP Request Method

  • GET : URL에 해당하는 자료의 전송을 요청, 클라이언트가 GET을 보내면 서버에서 그 주소에 해당하는 응답을 보내준다

  • POST : 서버가 처리할 수 있는 자료를 전송, 예를 들면 로그인할때 클라이언트가 폼에 아이디와 비밀번호를 입력하면 로그인 될 수 있게 해당되는 자료를 서버에서 처리한 후 결과를 전송해준다

  • PUT : 해당 URL에 자료를 저장

  • DELETE : 해당 URL의 자료를 삭제


HTTP Status Code

mdn참고하기

  • 200번대
  • 300번대
  • 400번대
  • 500번대


URL

URL은 서버에 자원(데이터)를 요청하기 위한 주소이다. URL은 URI의 상위개념으로 네트워크 상에서 자원 위치를 알려주기 위한 규약이다


CLI

기본적인 명령어

  • ls
  • cd
  • mkdir
  • open
  • pwd
  • rm

기초 명령어

  • cp : 복사 cp -r dir1 dir2 dir1안의 모든 내용을 dir2로 복사
  • mv : 이동 mv dir1 dir2 dir1안의 모든 내용을 dir2로 이동
  • ln : 하드링크, 소프트링크 생성 (정확히는…)
  • * : 와일드카드로 모든 문자 가리킴
  • less : 파일 내용을 보여준다
  • which(현재저장되어 있는 위치)
  • man(설명서)
  • alias(별칭 설정) alias django = 'cd/home/Desktop/django' .bashrc에 삽입후 source .bashrc로 업데이트
  • type

리다이렉션 명령어

  • ls -i /usr/bin>ls-out.txt : usr/bin 텍스트 내용을 out.txt에 전부 저장
  • cat :cat>파일명 입력받은 결과 지정한 파일로 복사
  • sort : 정렬
  • uniq : 중복제거
  • wc : 라인수 출력 (-l)
  • grep : 패턴에 일치하는 문자 출력
  • | : 필터사용이 가능 , 여러 명령어 동시 사용 가능
  • head/tail


경로

  • 절대경로 : 루트디렉토리에서 원하는 디렉토기까지의 경로
  • 상대경로 : 현재 작업 디렉토리가 시작점이 된다


가상환경(Virtual Environment)

내 컴퓨터 내에 특정 공간에서만 사용하도록 만드는 제한된 공간을 만드는 것으로 컴퓨터 내의 독립된 환경을 만들어 로컬 전역에 설치하지 않아도 패키지를 사용할 수 있다. 사용하는 이유는 내 컴퓨터에 설치되어있는 버전에 방해받지 않고 개발 환경을 맞출 수 있다는 장점이 있다


virtualenv 설치

  • pip3 install virtualenv
  • mkdir testproject
  • cd testproject
  • virtualenv venv : venv(=컨벤션)의 가상환경 설정
  • source venv/bin/activate : venv 가상환경 실행
  • pip install requests : 원하는 패키지 설치
  • pip freeze>requirements.txt : 패키지들의 버전이 명시된 txt 파일 생성됨
  • pip install -r requirements.txt : 명세를 기준으로 패키지들을 설치

import requests
from bs4 import BeautifulSoup

url = 'https://search.naver.com/search.naver?where=nexearch&sm=top_hty&fbm=1&ie=utf8&query=%EC%BD%94%EB%A1%9C%EB%82%98'
response = requests.get(url)

if response.status_code == 200:
html = response.text
soup = BeautifulSoup(html, 'html.parser')

ul = soup.find('ul',{'class':'list_news'}) # ul이 배열이 아니므로 ul안에 있는 li를 요소로 가지고 오는 배열을 find_all을 통해 가져온 후 lis에 넣어준다 그럼 lis를 for문을 돌려 print할 수 있다
lis = ul.find_all('li',{'class','bx'})

for li in lis:
print(li.find('a','news_tit').text)
print(li.find('a',{'class':'news_tit'}).text)
print(li.find('a',{'class':'news_tit'})['href'])
