form이란

form은 입력에 대한 처리를 할수 있는 역할을 해주는데 장고에 있는 form을 사용할 수 있으며 model 클래스와 유사하게 정의한다. 사용자로부터 정보를 수집하여 서버에 제출하는데 해당 값들에 대한 유효성 검증, 데이터 처리하는 로직등 복잡함을 단순화 시켜주고 자동화 기능을 제공해준다 .
html에 있는 form에 쓰이는 속성들에 대해서 먼저 알아보자

  • type 속성 : 어떤 종류의 위젯이 표시될지 정의한다. text, password, sumbit 등 어떤 동작을 하는지에 따라 다르다
  • submit : 폼이 제출될때 처리가 필요한 데이터를 전달해준다
  • name, id : javascript,html,css에 있는 필드를 확인하는데 사용된다
  • value : 필드가 처음 표시될 때의 초기값을 정의한다
  • label 태그 : 관련된 input등의 이름이 명시된다
  • for : 관련된 input의 id를 포함하고 있다
  • action : 폼이 제출 될때 처리가 필요하 데이터를 전달받는 곳의 url주소, 설정이 없다면 폼은 현재 페이지 url로 다시 제출된다 - method : POST, GET 두가지로 데이터를 보낼 수 있다.


forms.py 만들기

django 폼을 사용하는 앱에 forms.py 파일을 생성하여 login 폼을 만들어준다.
가장 먼저 forms model을 import 해주고 LoginForm이라는 login 폼을 만들어준다. 해당 클래스는 login.html 템플릿에서 데이터를 불러올때 사용이 되는 데이터다. username,password의 필드를 생성한다.

  • label : 화면에 나오는 name을 설정해준다
  • error_messages={‘required’:’’} : 화면에 보여줄 에러 메시지를 지정한다. required는 반드시 입력해야 하는 값이 입력되지 않았을때 발생하는 에러이다
  • widget = forms.PasswordInput : CharField는 은닉기능이 없으므로 PasswordInput이라는 위젯을 사용해 화면에 출력될때 **처리 될 수 있게 해준다
forms.py
from django import forms

class LoginForm(forms.Form): #2개의 필드를 사용하는 폼이 생성됨
    username = forms.CharField(
        error_messages={
            'required':'아이디를 입력해주세요'
        },
        max_length=32, label='사용자이름')
    password = forms.CharField(
        error_messages={
            'required':'비밀번호를 입력해주세요'
        },
        widget=forms.PasswordInput, label='비밀번호')


form 선언하기

✔ 앱안에 forms.py파일을 생성하자. form 데이터를 생성하기 위해서 forms라이브러리를 import하고 form클래스를 생성한다. LoginForm class를 생성하였다
사용자의 id와 pwd를 입력할 CharField 2개를 가지고 온다

forms.py

from django import forms

class LoginForm(forms.Form):
    username = forms.CharField(max_length=32)
    password = forms.CharField()


✔ 작성한 form 필드를 가져와 view에서 화면에 출력될 수 있게 작성한다
보통 view에서 POST 요청 타입인지 판단하여 요청 여부를 확인한다.

views.py

form안의 값 유효성 검증하기

  • cleaned_data = super().clean() : super를 통해서 기존에 form 안에 들어있던 clean함수를 먼저 호출해준다.
    그 후 검증이 끝난 username과 password 를 가져와준다

if문을 통해 username과 password가 들어있을때 조건문 처리를 해준다

  • fcuser = Fcuser.objects.get(username=username) : model fcuser에 있는 uername을 받아온다
  • if not check_password(password, fcuser.password): 입력받은 password와 model에 저장되어 있는 password 값이 같은지 확인하다
  • self.add_error(‘password’,’비밀번호를 틀렸습니다’) : form안에 기본적으로 error를 출력해주는 함수를 사용하기 위해 self를 작성하였다
  • self.user_id = fcuser.id : 비밀번호가 맞았을때 form 변수 안에 있는 user_id에 model에 있는 fcuser.id가 저장된다
forms.py
from django import forms
from .models import Fcuser
from django.contrib.auth.hashers import check_password

class LoginForm(forms.Form): 
    .
    .
    def clean(self):
        cleaned_data = super().clean()
        username = cleaned_data.get('username')
        password = cleaned_data.get('password')
        
         if username and password:
              fcuser = Fcuser.objects.get(username=username)
              if not check_password(password, fcuser.password):
                 self.add_error('password','비밀번호를 틀렸습니다')
              else:
                 self.user_id = fcuser.id


views.py에서 session을 작성해준다

  • request.session[‘user’] = form.user_id : 유효성 검사를 모두 통과하여 id와 pwd가 일치하기 때문에 세션의 값을 삽입한다
  • return redirect(‘/’) : home으로 화면이 전환된다
def login(request):  #template인 login.html과 연결시켜준다
    if request.method == "POST":
        form = LoginForm(request.POST)
        if form.is_valid():  # forms.py에서 검증을 하였다
                
           request.session['user'] = form.user_id  #froms.py에서 비밀번호가 맞았을경우 self.user_id = fcuser.id로 id를 저장하였다           

           return redirect('/')
    else:
        form = LoginForm()

    return render(request, 'login.html', {'form' : form})
